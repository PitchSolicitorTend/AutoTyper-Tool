# Contributing to AutoTyper Tool

First off, thank you for considering contributing to **AutoTyper Tool** — the free, open-source auto typer for Windows! 🎉

The following is a short set of guidelines to help you contribute effectively.

## Code of Conduct

Be respectful, constructive, and welcoming. Harassment, discrimination, or abusive behavior of any kind will not be tolerated.

## How Can I Contribute?

There are many ways to help, even without writing code:

- 🐛 Report bugs and reproduce issues
- 💡 Suggest new features or improvements
- 📝 Improve documentation (README, guides, FAQ)
- 🌍 Help translate the UI into other languages
- 🧪 Test pre-release builds and report feedback
- ⭐ Star the repo and share it with others

## Reporting Bugs

Before opening a new issue:

1. **Search existing issues** to avoid duplicates.
2. Make sure you're using the **latest release** (see [CHANGELOG.md](CHANGELOG.md)).

When reporting a bug, please include:

- Your Windows version (10 or 11, build number)
- AutoTyper Tool version
- Steps to reproduce the issue
- Expected vs. actual behavior
- Screenshots or a short screen recording, if possible

## Suggesting Features

Feature requests are tracked as GitHub Issues. Please describe:

- The problem your feature would solve
- How you imagine it working
- Any alternatives you've considered

## Pull Request Process

1. **Fork** the repository and create your branch from `main`:
   ```
   git checkout -b feature/your-feature-name
   ```
2. Make your changes, keeping commits focused and descriptive.
3. Update documentation (`README.md`, `docs/`, `CHANGELOG.md`) if your change affects usage or behavior.
4. Make sure your code builds and runs correctly on Windows 10/11.
5. Open a Pull Request with:
   - A clear title and description of the change
   - A reference to any related issue(s) (e.g. `Fixes #12`)
6. A maintainer will review your PR, may request changes, and will merge it once approved.

## Style Guide

- Keep functions small and focused.
- Prefer clear, descriptive names over abbreviations.
- Avoid adding new dependencies unless necessary — AutoTyper Tool aims to stay small and portable.
- Match the existing code formatting/style of the file you're editing.
- Document new settings, hotkeys, or config options in [docs/usage-guide.md](docs/usage-guide.md).
